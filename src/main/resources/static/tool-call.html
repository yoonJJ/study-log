[AI 개념 정리] Tool Call (도구 호출)

작성일: 2026-04-10 · AI 시스템 개념 노트

LLM은 텍스트 생성에 강하지만 실시간 조회, 코드 실행, 외부 시스템 조작은 스스로 할 수 없습니다. Tool Call은 LLM이 외부 기능을 요청하고 앱이 실행하도록 연결하는 메커니즘입니다.

핵심 한 줄: Tool Call은 LLM이 직접 실행하지 않고, 호출할 함수/인수를 구조화된 형태로 요청하게 만드는 기능입니다.

1. Tool Call이란?

  • 실시간 정보 조회 (예: 날씨)
  • 정확한 계산/코드 실행
  • 외부 시스템 조작 (메일/캘린더)
  • 내부 DB/업무 시스템 조회

핵심은 LLM은 함수 실행 주체가 아니라 호출 요청 생성 주체라는 점입니다.

2. Tool Call vs Function Calling

구분Function CallingTool Calling
범위커스텀 함수 중심함수 + 내장 도구 포함
용어초기 표현현재 표준 표현

3. 전체 실행 흐름

  1. 도구(JSON Schema) 정의
  2. LLM 호출 시 도구 목록 전달
  3. LLM이 tool_calls JSON 반환
  4. 앱이 실제 함수 실행
  5. 결과를 tool 메시지로 다시 전달
  6. LLM이 최종 자연어 답변 생성
# OpenAI tool 정의 예시
tools = [{
  "type": "function",
  "function": {
    "name": "get_weather",
    "description": "도시의 현재 날씨를 반환합니다.",
    "parameters": {
      "type": "object",
      "properties": { "city": {"type": "string"} },
      "required": ["city"]
    }
  }
}]
# LLM 응답 예시
{
  "tool_calls": [{
    "id": "call_abc123",
    "type": "function",
    "function": {
      "name": "get_weather",
      "arguments": "{\"city\":\"서울\"}"
    }
  }]
}

4. tool_choice 옵션

값동작
auto모델이 도구 사용 여부 판단
none도구 사용 금지
required도구 사용 강제
특정 함수 지정해당 함수 호출 강제

5. Parallel Tool Call

독립적인 요청(예: 서울/도쿄 날씨)을 한 번에 여러 개 호출해 응답 지연을 줄일 수 있습니다.

{
  "tool_calls": [
    {"id":"call_001","function":{"name":"get_weather","arguments":"{\"city\":\"서울\"}"}},
    {"id":"call_002","function":{"name":"get_weather","arguments":"{\"city\":\"도쿄\"}"}}
  ]
}

7. Tool Call과 MCP 관계

  • Tool Call: 무엇을 호출할지 결정
  • MCP: 외부 서비스 연결 방식을 표준화

8. 실전 패턴: Agentic Loop

while True:
  response = llm(messages, tools)
  if not response.tool_calls:
    return response.text
  for tc in response.tool_calls:
    result = run_tool(tc)
    messages.append(tool_result(tc.id, result))

9. 활용 사례

  • 실시간 정보 조회
  • 계산/코드 실행
  • DB 조회
  • 외부 서비스 액션
  • 파일 처리/문서 자동화

10. 주의점

  • 도구 설명은 구체적으로 작성
  • 도구 개수는 최소화
  • 오류 처리 필수
  • 파괴적 액션은 Human-in-the-loop 적용

11. 정리

Tool Call은 LLM을 “텍스트 생성기”에서 “실행 가능한 에이전트”로 확장시키는 핵심입니다.
